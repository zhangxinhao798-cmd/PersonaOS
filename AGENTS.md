# AGENTS.md

## Project Context

PersonaOS is a long-term AI operating system project. Treat the repository as an evolving system that should remain understandable, modular, and easy to extend over time.

## Repository Rules

- Never modify `README.md` unless explicitly requested.
- Never rename folders.
- Never delete files without permission.
- Prefer documentation before implementation.
- Use small commits only.
- Ask before introducing new dependencies.
- Keep architecture modular.
- Prioritize the Python backend first.
- Defer frontend work until later unless explicitly requested.
- Always explain major changes before editing.

## Working Style

- Make focused, minimal changes.
- Preserve existing structure and naming.
- Document intent before adding complex behavior.
- Favor clear interfaces between modules.
- Treat destructive actions as permission-only operations.
